Changing layout files.

To change the page wireframe, modify the page layout files; all other customizations are performed in the page configuration or generic layout files.
Чтобы изменить каркас страницы, измените файлы макета страницы; все остальные настройки выполняются в файлах конфигурации страницы или общего макета.
Manage layouts
Управлять макетами
To make layout changes available on every page, modify the default.xml file. For example, layout changes added to app/code/Vendor/Module/view/frontend/layout/default.xml are loaded on all pages. To add layout changes to a specific page, use a layout file that corresponds to the page’s path. For example, changes to the app/code/Vendor/Module/view/frontend/layout/catalog_product_view.xml page are loaded on the product details page.
Чтобы изменения макета были доступны на каждой странице, измените файл default.xml. Например, изменения макета, добавленные в app / code / Vendor / Module / view / frontend / layout / default.xml, загружаются на все страницы. Чтобы добавить изменения макета на определенную страницу, используйте файл макета, соответствующий пути к странице. Например, изменения на странице app / code / Vendor / Module / view / frontend / layout / catalog_product_view.xml загружаются на страницу сведений о продукте.

block
блок
Defines a block.
Определяет блок.
Details: A block is a unit of page output that renders some distinctive content (anything visually tangible for the end-user), such as a piece of information or a user interface element.
Подробности: блок - это единица вывода страницы, которая отображает некоторый отличительный контент (что-либо визуально осязаемое для конечного пользователя), например, часть информации или элемент пользовательского интерфейса.
Blocks are a foundational building unit for layouts in Magento. They are the link between a PHP block class (which contains logic) and a template (which renders content). Blocks can have children and grandchildren (and so on). Information can be passed from layout XML files to blocks using the child node.
Блоки - это фундаментальная единица построения макетов в Magento. Они являются связующим звеном между классом блока PHP (который содержит логику) и шаблоном (который отображает контент). У блоков могут быть дети и внуки (и так далее). Информация может передаваться из XML-файлов макета в блоки с помощью дочернего узла .
Blocks employ templates to generate HTML. Examples of blocks include a category list, a mini cart, product tags, and product listing.
Блоки используют шаблоны для генерации HTML. Примеры блоков включают список категорий, мини-корзину, теги продуктов и список продуктов.
We recommend always adding a name to blocks. Otherwise, it is given a random name.
Мы рекомендуем всегда добавлять имя к блокам. В противном случае ему дается случайное имя.
Attribute Description Values Required?

container

A structure without content that holds other layout elements such as blocks and containers.
Структура без содержимого, содержащая другие элементы макета, такие как блоки и контейнеры.
Details: A container renders child elements during view output generation. It can be empty or it can contain an arbitrary set of <container> and <block> elements. If the <container> is empty, and there is no child <block> available, it will not be displayed in the frontend source code.
Подробности: контейнер отображает дочерние элементы во время генерации вывода представления. Он может быть пустым или содержать произвольный набор элементов <container> и <block>. Если <container> пуст и нет доступного дочернего <block>, он не будет отображаться в исходном коде внешнего интерфейса.
We recommend always adding a name to containers. Otherwise, it is given a random name.
Мы рекомендуем всегда добавлять имя к контейнерам. В противном случае ему дается случайное имя.
Attribute Description Values Required?
name A name that can be used to address the container in which this attribute is assigned. The name must be unique per generated page. If not specified, it will be auto-generated. A-Z, a-z, 0-9, underscore (_), period (.), dash (-). Should start with a letter. Case-sensitive. No
label Describes the purpose of the container. Any No
before Used to position the container before an element under the same parent. The element name or alias name is specified in the value. Use dash (-) to position the block before all other elements of its level of nesting. See before and after attributes for details. Element name or dash (-) No
after Used to position the container after an element under the same parent. The element name or alias name is specified in the value. Use dash (-) to position the block after all other elements of its level of nesting. See before and after attributes for details. Element name or dash (-). No

Controlling children visibility
Контроль над видимостью детей
The output attribute controls the visibility of the container’s children elements. Set this value to 1 to render children content or 0 to disable the output of the entire container.
Атрибут output управляет видимостью дочерних элементов контейнера. Установите для этого значения 1, чтобы отображать дочерний контент, или 0, чтобы отключить вывод всего контейнера.
Use this feature to make temporary changes to a store, such as disabling a section of the page for a sales event and re-enabling it after the event ends.
Используйте эту функцию, чтобы внести временные изменения в магазин, например отключить раздел страницы для события продаж и повторно включить его после завершения мероприятия.

block vs. container

Blocks represents the end of the chain in rendering HTML for Magento.
Блоки представляют собой конец цепочки рендеринга HTML для Magento.
Containers contain blocks and can wrap them in an HTML tag.
Контейнеры содержат блоки и могут заключать их в теги HTML.
Containers do not render any output if there are no children assigned to them.
Контейнеры не отображают никаких выходных данных, если им не назначены дочерние элементы.

before and after attributes

To help you to position elements in a specific order suitable for design, SEO, usability, or other requirements, Magento software provides the before and after layout attributes. These optional attributes can be used in layout XML files to control the order of elements in their common parent.
Чтобы помочь вам расположить элементы в определенном порядке, подходящем для дизайна, SEO, удобства использования или других требований, программное обеспечение Magento предоставляет атрибуты макета до и после. Эти необязательные атрибуты можно использовать в XML-файлах макета для управления порядком элементов в их общем родительском элементе.
The following tables give a detailed description of the results you can get using the before and after attributes. The first table uses a block a as positioned element.
В следующих таблицах дается подробное описание результатов, которые можно получить с помощью атрибутов до и после. В первой таблице в качестве позиционированного элемента используется блок a.
Attribute Value Description
before Dash (-) The block displays before all other elements in its parent node.
before [element name] The block displays before the named element.
before Empty value or [element name] is absent Use the value of after. If that value is empty or absent as well, the element is considered as non-positioned.
after Dash (-) The block displays after all other elements in its parent node.
after [element name] The block displays after the named element.
after Empty value or [element name] is absent Use the value of before. If that value is empty or absent as well, the block is considered as non-positioned.

Examples

Situation Result
Both before and after attributes are present after takes precedence.
Both before and after attributes are absent or empty The element is considered as non-positioned. All other elements are positioned at their specified locations. The non-positioned element displays at a random position that doesn’t violate requirements for the positioned elements.
Several elements have before or after set to dash (-) All elements display at the top (or bottom, in case of the after attribute), but the ordering of group of these elements is undefined.
The before or after attribute’s value refers to an element that is not located in the parent node of the element being defined. The element displays at a random location that doesn’t violate requirements for the correctly positioned elements.

action

The <action> instruction is deprecated. If the method implementation allows, use the <argument> for <block> or <referenceBlock> to access the block public API.
Тег <action> инструкция устарела. Если реализация метода позволяет, используйте <argument> для <block> или <referenceBlock> для доступа к публичному API блока.
Calls public methods on the block API.
Вызывает общедоступные методы в API блока.
Details: Used to set up the execution of a certain method of the block during block generation; the <action> node must be located in the scope of the <block> node.
Подробности: используется для настройки выполнения определенного метода блока при генерации блока; тег <action> узел должен находиться в области действия <block> узел.
Text true
"action" child nodes are translated into block method arguments. Child nodes names are arbitrary. If there are two or more nodes with the same name under "action", they are passed as one array.
Дочерние узлы "action" преобразуются в аргументы блочного метода. Имена дочерних узлов произвольны. Если в "action" есть два или более узлов с одинаковым именем, они передаются как один массив.
Attribute Description Values Required?
method The public method that is called during block generation. The method name in the block yes
To pass parameters, use the (argument)(/argument) instruction.
Для передачи параметров используйте инструкцию (аргумент) (/ аргумент).

referenceBlock and referenceContainer

Updates in (referenceBlock) and (referenceContainer) are applied to the corresponding (block) or (container).
Обновления в (referenceBlock) и (referenceContainer) применяются к соответствующему (блоку) или (контейнеру).
For example, if you make a reference by (referenceBlock name="right"), you are targeting the block (block name="right").
Например, если вы делаете ссылку с помощью (referenceBlock name = "right"), вы нацеливаетесь на блок (block name = "right").
To pass parameters to a block use the (argument)(/argument) instruction.
Чтобы передать параметры блоку, используйте инструкцию (аргумент) (/ аргумент).
Attribute Description Values Required?
remove Allows to remove or cancel the removal of the element. When a container is removed, its child elements are removed as well. true or false no
display Allows you to disable rendering of specific block or container with all its children (both set directly and by reference). The PHP objects of the block or container and its children are still generated and available for manipulation. true or false no
