Override a layout
Переопределить макет
What’s in this topic
Что в этой теме
Not all layout customizations can be performed by extending layouts. If the amount of customizations is large, you can use the overriding function for the needed layout file. This means that the new file that you place in the theme will be used instead of the parent theme layout file of base layout file.
Не все настройки макета можно выполнить путем расширения макетов. Если количество настроек велико, вы можете использовать функцию переопределения для необходимого файла макета. Это означает, что новый файл, который вы помещаете в тему, будет использоваться вместо файла макета родительской темы файла базового макета.
In this article, page layouts, page configurations, and generic layouts are referred to as layout files, as the mechanism of overriding is similar for all of them.
В этой статье макеты страниц, конфигурации страниц и общие макеты называются файлами макетов, поскольку механизм переопределения одинаков для всех из них.
Layout files with instructions that override the default or parent theme files are referred to as overriding layout files.
Файлы макета с инструкциями, которые заменяют файлы темы по умолчанию или родительские файлы, называются файлами макета замещающих файлов.
Examples of customizations that involve overriding layouts
Примеры настроек, включающих переопределение макетов
Examples of customizations that involve overriding layouts:
Примеры настроек, включающих переопределение макетов:
*Suppressing method invocation.
*Подавление вызова метода.
**Overriding is not necessary if a block has a method that cancels the effect of the originally invoked method. In this case, you can customize the layout by adding a layout file where the canceling method is invoked.
**В переопределении нет необходимости, если в блоке есть метод, который отменяет эффект первоначально вызванного метода. В этом случае вы можете настроить макет, добавив файл макета, в котором вызывается метод отмены.
*Modifying method arguments.
*Изменение аргументов метода.
*Canceling block/container removal using the remove attribute.
*Отмена удаления блока / контейнера с помощью атрибута remove.
*Setting XML attributes of blocks and containers.
*Установка атрибутов XML блоков и контейнеров.
Certain attributes, like htmlClass, htmlId, label attributes can be changed in extending layouts.
Некоторые атрибуты, такие как htmlClass, htmlId, атрибуты метки, можно изменить при расширении макетов.
*Removing block arguments.
*Удаление аргументов блока.
*Modifying and suppressing handles inclusion.
*Изменение и подавление включения дескрипторов.
*Removing all handle instructions by declaring an overriding layout file with an empty handle.
*Удаление всех инструкций дескриптора путем объявления замещающего файла макета с пустым дескриптором.

Override base layouts
Переопределить базовые макеты
To add an overriding base layout file (to override a base layout provided by the module):
Чтобы добавить замещающий файл базового макета (чтобы переопределить базовый макет, предоставленный модулем):
Put a layout file with the same name in the following location:
Поместите файл макета с тем же именем в следующее место:
 <theme_dir>
   |__/<Namespace_Module>
     |__/layout
       |__/override
          |__/base
            |--<layout1>.xml
            |--<layout2>.xml
These files override the following layouts:
Эти файлы переопределяют следующие макеты:
  • <module_dir>/view/frontend/layout/<layout1>.xml
  • <module_dir>/view/frontend/layout/<layout2>.xml

For example, <theme_dir>/Magento_Checkout/layout/override/base/checkout_cart_index.xml will override Magento_Checkout/view/frontend/layout/checkout_cart_index.xml.

Override theme layouts
Переопределить макеты темы
To add an overriding theme file (to override a parent theme layout):
Чтобы добавить замещающий файл темы (чтобы переопределить макет родительской темы):
Put a layout file with the same name in the following location:
Поместите файл макета с тем же именем в следующее место:
<theme_dir>
  |__/<Namespace_Module>
    |__/layout
      |__/override
         |__/theme
            |__/<Parent_Vendor>
               |__/<parent_theme>
                  |--<layout1>.xml
                  |--<layout2>.xml
These files override the following layouts:
Эти файлы переопределяют следующие макеты:
  • <parent_theme_dir>/<Namespace>_<Module>/layout/<layout1>.xml
  • <parent_theme_dir>/<Namespace>_<Module>/layout/<layout2>.xml

For example, <theme_dir>/Magento_Checkout/layout/override/theme/Magento/luma/checkout_cart_index.xml will override app/design/frontend/Magento/luma/Magento_Checkout/layout/checkout_cart_index.xml.

To override page layout files, use the page_layout directory name instead of layout.
Чтобы переопределить файлы макета страницы, используйте имя каталога page_layout вместо layout.
Customization mistakes
Ошибки настройки
Although the layout overriding mechanism provides great customization flexibility, it’s possible to use it to add logically irrelevant changes. We strongly recommend you not make the following changes:
Хотя механизм переопределения макета обеспечивает большую гибкость настройки, его можно использовать для добавления логически нерелевантных изменений. Мы настоятельно рекомендуем не вносить следующие изменения:
Changing block name or alias. The name of a block should not be changed, and neither should the alias of a block remaining in the same parent element.
Изменение имени или псевдонима блока. Имя блока не должно изменяться, равно как и псевдоним блока, оставшегося в том же родительском элементе.
Changing handle inheritance. For example, you should not change the page type parent handle.
Изменение наследования дескрипторов. Например, вы не должны изменять родительский дескриптор типа страницы.
