What is a Magento 2 Module?
Что такое модуль Magento 2?

A Module in Magento 2 is an independent component or a set of components that provide business logic and features. Magento 2 Open Source is a modular application and it supports modularity. It means that all functionality is implemented and delivered in components that are known as Modules. A Component is also known and called an Extension. A module and an extension is basically the same thing.
Модуль в Magento 2 - это независимый компонент или набор компонентов, которые предоставляют бизнес-логику и функции. Magento 2 Open Source - это модульное приложение, поддерживающее модульность. Это означает, что все функциональные возможности реализованы и поставляются в компонентах, известных как модули. Компонент также известен и называется расширением. Модуль и расширение - это в основном одно и то же.

Magento 2 Modules
Модули Magento 2

Each extension or module in Magento 2 provides a set of capabilities that support the business and functional logic of a module. It allows bringing additional features for the e-commerce store which is based on Magento 2.
Каждое расширение или модуль в Magento 2 предоставляет набор возможностей, которые поддерживают бизнес-логику и функциональную логику модуля. Это позволяет добавить дополнительные функции для магазина электронной коммерции, основанного на Magento 2.

Example
Пример

Let's take a Catalog module for example. The Catalog module provides a different set of features which include Category Management, Product Management, Product Types support for Simple and Virtual products, Catalog browsing capability and so on.
Возьмем, к примеру, модуль Каталога. Модуль «Каталог» предоставляет другой набор функций, включая управление категориями, управление продуктами, поддержку типов продуктов для простых и виртуальных продуктов, возможность просмотра каталога и т. Д.

Also, the module provides extensibility points for other modules which allow to extend the Catalog module with additional functionality. An example of such an additional module is a Downloadable Product module.
Кроме того, модуль предоставляет точки расширения для других модулей, которые позволяют расширить модуль Каталога дополнительными функциями. Примером такого дополнительного модуля является модуль Загружаемый продукт.

A module in Magento 2 could bring a complex set of features from one side and from the other side it could be a tiny feature that adds an icon to a Product page for example.
Модуль в Magento 2 может иметь сложный набор функций с одной стороны, а с другой стороны это может быть крошечная функция, которая, например, добавляет значок на страницу продукта.

A Module Name
Имя модуля

Each module's name consists of two parts. A vendor name and the name of a module. These two parts are concatenated with a _(underscore) or a \(backslash). As a result, the module name should follow the pattern VendorName_ModuleName. For example, a Catalog module name is going to be the name of a Magentovendor name and a Catalogmodule name, which is a Magento_Catalogor Magento\Catalog.
Название каждого модуля состоит из двух частей. Имя поставщика и имя модуля. Эти две части объединяются знаком _ (подчеркивание) или \ (обратная косая черта). В результате имя модуля должно соответствовать шаблону VendorName_ModuleName. Например, имя модуля Каталога будет именем имени Magentovendor и именем модуля Каталога, которым является Magento_Catalogor Magento \ Catalog.

Where should a Module be located in Magento 2?
Где должен быть расположен модуль в Magento 2?

That's a very interesting question. There are two locations where a module can be located in a Magento 2 application.
Это очень интересный вопрос. Есть два места, где модуль может быть расположен в приложении Magento 2.

Magento 2 Modules
Magento 2 Modules

The first location is an app/codedirectory. This directory is used to add all custom and 3rd-party Magento 2 modules. This location is usually used by development agencies, internal or in-house developers in order to simplify a development process.
Первое место - это каталог app / codedirectory. Этот каталог используется для добавления всех пользовательских и сторонних модулей Magento 2. Это местоположение обычно используется агентствами по развитию, внутренними или внутренними разработчиками, чтобы упростить процесс разработки.

The second location is a vendordirectory. This directory is used during the composer package management installation. A module installation in Magento 2 with a composer package manager is the recommended way of installing a Magento 2 module.
Второе место - это каталог поставщиков. Этот каталог используется во время установки управления пакетами композитора. Установка модуля в Magento 2 с помощью диспетчера пакетов композитора - это рекомендуемый способ установки модуля Magento 2.

Learn more about Magento 2 Open Source directory structure in the Project Structure Overview lesson.
Узнайте больше о структуре каталогов Magento 2 с открытым исходным кодом в уроке «Обзор структуры проекта».

How to create a Module?
Как создать модуль?

The first thing we have to do in order to create a module is to create a module directory. Usually, all modules are located under the app/codedirectory. If you follow the previous lesson, the codedirectory should be missing under the appdirectory of a Magento 2 project.
Первое, что нам нужно сделать, чтобы создать модуль, - это создать каталог модуля. Обычно все модули находятся в каталоге app / codedirectory. Если вы следуете предыдущему уроку, каталог codedirectory должен отсутствовать в appdirectory проекта Magento 2.

Inside the appdirectory, let's create the codedirectory. This is a directory where all modules should be located. Every Magento 2 module should be located under the app/code/VendorName/ModuleNamedirectory location.
Внутри appdirectory давайте создадим codedirectory. Это каталог, в котором должны находиться все модули. Каждый модуль Magento 2 должен находиться в папке app / code / VendorName / ModuleNamedirectory.

Let's create a vendor directory. In this lesson, the vendor is a MageMastery. Inside the app/code/MageMasterydirectory, let's create a module directory. For the module name, this is going to be a FirstModuledirectory name.
Создадим каталог поставщиков. В этом уроке продавец - MageMaster. Внутри каталога app / code / MageMasterydirectory создадим каталог модуля. Для имени модуля это будет имя FirstModuledirectory.

There are two required files that have to be created in each Magento 2 module. These files are used to notify to Magento 2 about a module. The registration.phpand module.xmlfiles.
В каждом модуле Magento 2 необходимо создать два обязательных файла. Эти файлы используются для уведомления Magento 2 о модуле. Модуль registration.phpand.xmlfiles.

Module Required Files
Необходимые файлы для модуля

The registration.php file
Файл registration.php

The registration.phpfile is used to notify to Magento 2 application about a module in a file system. It also used to provide a location to a Magento 2 module. Place the new registration.phpfile inside the app/code/MageMastery/FirstModuledirectory.
Файл registration.php используется для уведомления приложения Magento 2 о модуле в файловой системе. Он также используется для указания местоположения модуля Magento 2. Поместите новый файл registration.php в папку app / code / MageMaster / FirstModuledirectory.

The content of the file is as follows:
Содержимое файла выглядит следующим образом:

<?php

use Magento\Framework\Component\ComponentRegistrar;

ComponentRegistrar::register(
    ComponentRegistrar::MODULE,
    'MageMastery_FirstModule',
    __DIR__
);
The Magento\Framework\Component\ComponentRegistrarclass is used to register a new module in a Magento 2 application.
Класс Magento \ Framework \ Component \ ComponentRegistrar используется для регистрации нового модуля в приложении Magento 2.

The static register()method accepts three arguments. The $componentTypeargument accepts a type of a component, which in case of a module is ComponentRegistrar::MODULE. The ComponentRegistrar::MODULEis a constant of the Magento\Framework\Component\ComponentRegistrarclass and it is recommended to use this constant for the module registration.
Статический метод register () принимает три аргумента. Параметр $ componentTypeargument принимает тип компонента, которым в случае модуля является ComponentRegistrar :: MODULE. ComponentRegistrar :: MODULE - это константа класса Magento \ Framework \ Component \ ComponentRegistrar, и рекомендуется использовать эту константу для регистрации модуля.

The second argument of the register()method is the name of the module. As you can see the name is a VendorName_ModuleName. In the above code sample, the name is MageMastery_FirstModule.
Второй аргумент метода register () - это имя модуля. Как видите, имя - VendorName_ModuleName. В приведенном выше примере кода имя MageMaster_FirstModule.

The final argument is a PHP __DIR__constant. It provides an actual location of the module and is interpreted into an /var/www/magento/app/code/MageMastery/FirstModulein case a Magento 2 application is installed in the /var/www/magentodirectory.
Последний аргумент - это PHP __DIR__constant. Он предоставляет фактическое местоположение модуля и интерпретируется в / var / www / magento / app / code / MageMaster / FirstModule в случае, если приложение Magento 2 установлено в каталоге / var / www / magentodirectory.

How does this work?
Как это работает?

One of the allowed locations of the registration.phpfile is app/code/VendorName/ModuleNamedirectory (in case of the current lesson the location is app/code/MageMastery/FirstModule/). The registration.phpfile is scanned and executed during the Magento 2 Open Source application start. The method register()of the ComponentRegistrarclass adds the module name and its location into the global pool of all module's locations. Once added, Magento 2 Open Source then goes through all registered modules paths and reads the configuration of the module.xmlfile.
Одно из разрешенных мест для файла registration.php - app / code / VendorName / ModuleNamedirectory (в случае текущего урока это app / code / MageMaster / FirstModule /). Файл registration.php сканируется и запускается при запуске приложения Magento 2 с открытым исходным кодом. Метод register () класса ComponentRegistrar добавляет имя модуля и его расположение в глобальный пул всех расположений модуля. После добавления Magento 2 с открытым исходным кодом проходит по всем зарегистрированным путям модулей и считывает конфигурацию файла module.xmlfile.

That is why it is important to have both files to successfully register a Module in Magento 2.
Вот почему важно иметь оба файла для успешной регистрации модуля в Magento 2.

The module.xml file
Файл module.xml

The module.xmlfile is used to declare and provide information about the module name and any dependencies to other 3rd-party or out-of-box Magento 2 modules.
Файл module.xmlfile используется для объявления и предоставления информации об имени модуля и любых зависимостях от других сторонних или готовых модулей Magento 2.

The module.xmlfile is a main module's configuration file. It is required to place this file into the module's etcdirectory. Let's create the directory inside the app/code/MageMastery/FirstModuledirectory module.
Module.xmlfile - это главный файл конфигурации модуля. Этот файл необходимо поместить в каталог модуля etc. Создадим каталог внутри модуля app / code / MageMaster / FirstModuledirectory.

An example of the XML configuration file is as follow:
Пример файла конфигурации XML выглядит следующим образом:

<?xml version="1.0"?>
<config>

</config>
This is a basic file content that is used across almost all Magento 2 configuration files. We are going to learn more about XML configuration files in the next lessons.
Это базовое содержимое файла, которое используется почти во всех файлах конфигурации Magento 2. В следующих уроках мы узнаем больше о файлах конфигурации XML.

Additionally, all XML files in Magento 2 contain 2 attributes in the node. The xmlnsattribute provides a location to the XMLSchema instance and the xsi:noNamespaceSchemaLocationattribute provides a path to an XSD (XML Schema Definition) file of a Magento 2 framework. The last one is used to provide XML validation and syntax highlighting capability to a configuration file.
Кроме того, все файлы XML в Magento 2 содержат 2 атрибута в узле . Атрибут xmlnsattribute обеспечивает расположение экземпляра XMLSchema, а атрибут xsi: noNamespaceSchemaLocation предоставляет путь к файлу XSD (определение схемы XML) фреймворка Magento 2. Последний используется для обеспечения возможности проверки XML и выделения синтаксиса для файла конфигурации.

These attributes are not required, however, it is recommended to follow the standard Magento practices and include it into XML files.
Эти атрибуты не требуются, однако рекомендуется следовать стандартным методам Magento и включать их в файлы XML.

As a result, the final module.xmlfile should look like follow:
В результате финальный файл module.xmlfile должен выглядеть следующим образом:

<?xml version="1.0"?>
<config xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" 
        xsi:noNamespaceSchemaLocation="urn:magento:framework:Module/etc/module.xsd">
    <module name="MageMastery_FirstModule" />
</config>
The only required XML node which has to be added is a <module>one. The nameattribute should include the name of the current module, which is MageMastery_FirstModule.
Единственный требуемый узел XML, который нужно добавить, - это узел <module>. Атрибут name должен включать имя текущего модуля, которым является MageMaster_FirstModule.

How to register a Module?
Как зарегистрировать модуль?

The last step is to run the command which will enable and install the MageMastery_Firstmodule.
Последний шаг - запустить команду, которая включит и установит модуль MageMaster_Firstmodule.
bin/magento setup:upgrade
You may notice a new module added to the list of modules.
Вы можете заметить, что в список модулей добавлен новый модуль.

In order to check if the module has been added and enabled in a Magento 2 application. Here I use a grepcommand to check a string in an app/etc/config.phpfile.
Чтобы проверить, был ли модуль добавлен и включен в приложении Magento 2. Здесь я использую команду grep для проверки строки в файле app / etc / config.phpfile.

grep MageMastery_FirstModule app/etc/config.php
As a result, a terminal screen should show a
В результате на экране терминала должен отображаться

The value 1 means that the module is enabled, and 0 means that the module is disabled. As we can see from the output, the MageMastery_FirstModuleis set to 1, which means that the module has been successfully registered in a Magento 2 Open Source application.
Значение 1 означает, что модуль включен, а значение 0 означает, что модуль отключен. Как видно из выходных данных, для MageMaster_FirstModule установлено значение 1, что означает, что модуль успешно зарегистрирован в приложении Magento 2 с открытым исходным кодом.
